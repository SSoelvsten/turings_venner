Abstract

The Anatree (Anagram Tree) allows one to solve constraint satisfaction problems where the order of the symbols in each input word does not matter. What makes it unique is that the complexity of each operation on this data structure only depends on the number of symbols in the alphabet and the length of each word rather than the total number of words being processed.
I want to show the fundamental design of the Anatree, the basic recursive operations on it, and finally show how it is used in practice on a project of mine.
